# Agent HQ - Hooks 設定檢測功能設計

## 需求背景

確保 Agent 追蹤功能正常運作，需要 Claude Code Hooks 正確設定。

---

## UI 設計

### 1. 檢測時機

**服務啟動時自動檢測**：
- 後端啟動時檢測 `~/.claude/settings.json` 中是否有 hooks 設定
- 前端載入時顯示檢測結果

### 2. 檢測結果狀態

| 狀態 | 圖示 | 顏色 | 顯示內容 |
|------|------|------|----------|
| **已設定** | ✅ | 綠色 | "Hooks 已正確設定" |
| **未設定** | ⚠️ | 黃色 | "Hooks 未設定，部分功能可能無法使用" |
| **設定失敗** | ❌ | 紅色 | "Hooks 設定失敗，請手動設定" |

### 3. 顯示位置

**方案 A：Status Bar 內**
```
[Project: agent-hq] [● Connected] [85 agents] [⚠️ Hooks]
```

點擊 ⚠️ 圖示可展開設定提示。

**方案 B：Header 專用區域（推薦）**
```
┌─────────────────────────────────────────────────────────────┐
│ ◉ Agent HQ  │  agent-hq ▼  │  [-] [50%] [⛶]  │ [⚠️ Hooks]  │
└─────────────────────────────────────────────────────────────┘
```

點擊後彈出 modal。

## 實作細節

### 後端 API

```javascript
// GET /api/hooks/status
{
  "configured": true/false,
  "events": ["SubagentStart", "SubagentStop"],
  "hookScriptExists": true/false
}
```

### 前端流程

1. **App 載入** → call `/api/hooks/status`
2. **若未設定** → 顯示 ⚠️ icon in header
3. **點擊 icon** → 顯示 Modal
4. **點擊一鍵設定** → call `/api/hooks/setup`
5. **設定成功** → 顯示成功訊息 + 提醒重啟 Claude Code

### 一鍵設定功能

後端執行：
1. 建立 `~/.claude/hooks/send-hook.sh`（若不存在）
2. 更新 `~/.claude/settings.json` 加入 hooks 設定
3. 備份原設定檔

- **顏色**：與現有一致（深色主題）
- **按鈕**：

- **動畫**：Modal 淡入效果 (200ms)
